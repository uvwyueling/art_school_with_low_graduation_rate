## 3. ETL总结
**处理逻辑：**
* 统一对四个表格进行了多维清洗（清理字符串空格、剔除非设计类的学生数据、逐一调查和处理缺失值、确认无重复记录、数据类型转换、学号脱敏）
* 针对清洗过程中输出的缺失值及异常情况逐一进行穿透式探查、修复
* 清洗后的映射关系及具体情况如下： 

| 原表名称 | Dataframe名称 | 异常情况的处理备注 | 是否有新增列 |
| :--- | :--- | :--- | :--- |
| `stage_reassign.csv` | `df_stg_reassign_ready` | 有三位同学在公布最终专业前申请休学导致‘最终专业’缺失值。他们在一年后复学分别加入2018级和2019级，不属于本次项目的分析对象，因此剔除他们的专业分流志愿数据。因为他们复学后不属于2014-2017级学生，所以其他三张表中均不包含他们的数据。 | 有，通过逻辑判断（'拟分流专业第一志愿'和'最终专业'两个字段是否吻合）新增 [is_reassign] 列 (True, False) | 
| `raw_2014_first_year_GPA.csv` | `df_2014_1y_gpa_ready` | 未发现异常情况 | 否 |
| `raw_graduation.csv`| `df_raw_grad_ready` | 探查发现某位同学的[班级]缺失值，进入教务系统寻找后修复 | 有，通过逻辑判断（'毕业年份'数字减去'年级'数字得到 4、5、6、'毕业年份'缺失值，四种情况）新增 [grad_status] 列('四年正常'、'五年异常'、'六年异常'、'未毕业') |
| `raw_last3y_gpa.csv` | `df_raw_last3y_gpa_ready` | 未发现异常情况 | 否 |
