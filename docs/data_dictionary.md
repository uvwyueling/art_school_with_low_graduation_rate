**表名：** `to_verify_17students.csv`
**说明：** 该表格由 `./scripts/special_student_check.py` 脚本生成前五列，后面三列数据经由教务系统探查后手工填入。学校教务系统会默认给所有休学、降级学生的 '年级' 字段+1，因此清洗后分析宽表 `dim_student` 中的[grad_status]列（意图体现学生的修读年限以及最后是否毕业，由[毕业年份]列 - [年级]列得到）对于降级学生来说不再准确。（降级同学的修学时长往往是五年或六年）

下面通过举例来说明这个表格存在的必要性：<br>
1️⃣ 【常规学生】学号2013062301 的学生是2013年入学的、2013级，毕业于2017。按照 'grad_status' 列计算逻辑，2017-2013 = ‘四年毕业’；<br>
2️⃣ 【休学、复学学生】学号2013062301 的学生在大学期间休学一年后再复学，'年级' 字段被教务处调整为2014级，毕业于2018。按照 'grad_status' 列计算逻辑，2018-2014 = ‘四年毕业’，也是符合实际情况的，因为该同学中间一年休学，实际在校学习的时间就是四年；<br>
3️⃣ 【降级学生】学号2013062301 的学生在大四下学期发现学分修不满，无法毕业，只能降级再读一年，于是'年级' 字段被教务处调整为2014级，假设该同学在下一学年把剩下的学分修满，于2018年毕业。按照 'grad_status' 列计算逻辑，2018-2014 = ‘四年毕业’，这与实际情况不符，因为他的总成绩单上显示10个学期的课程名称与分数，所以这位同学应该属于’五年异常’。<br>

**处理逻辑：**
* 根据表格前部的信息手工从教务系统找到每位同学
* 写入[专业]和（学籍异动的）[真实情况] 字段
* 筛选同时符合 "设计专业" 和 "降级" 这两个条件的学生
* 新建一列，在符合筛选条件的同学所在的行写入 '是' 完成标记

| :--- | :--- | :--- | :--- |
| 学号 | INTEGER | 学生唯一ID | 主键，无缺失 |
| 姓名 | TEXT | 学生姓名 | 无缺失 |
| 年级 | INTEGER | 教务系统导出的‘年级’字段 | 无缺失，常规情况下学生的“入学年份”=“年级”。但是如果学生休学、降级，则“年级”字段与“入学年份”字段不相关 |

* 针对清洗过程中输出的缺失值及异常情况逐一进行穿透式探查、修复
* 清洗后的映射关系及具体情况如下：
